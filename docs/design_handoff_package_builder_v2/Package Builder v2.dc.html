Aasan Packages v2 — prototype

The new model — entitlement blocks

A package is a stack of blocks instead of a flat service list: Fixed items, Choice groups (pick N — locked at purchase or chosen each visit), Unlimited, Session pools and Credit. Any block can be a free bonus. This expresses everything from "10 haircuts" to "3 facials of choice + unlimited threading".

Selling — recommendation

Today, tapping a package card in POS is an unwired TODO that would add a bill line directly, bypassing the cart. Instead: a Configure & Sell sheet binds the customer, resolves purchase-time choices, previews expiry — then drops one package line into the existing cart. One path for discounts, refunds and receipts. The cart itself is unchanged.

What stays

The redemption side is already good: entitlements rail, FIFO multi-package selector, auto-apply, gold redemption lines with undo. Kept as-is — see the Customer view tab. Pricing keeps your model: sum of value minus a discount (%, ₹ off, or final price).
