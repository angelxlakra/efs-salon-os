# V2 Phase 4 — POS Retrofit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring all POS page components into full V2 token compliance — replacing raw Tailwind color literals with semantic design tokens and upgrading spinner/bespoke loading+empty states to the Skeleton and EmptyState primitives.

**Architecture:** Five independent component retrofits applied in dependency order (page shell → service picker → cart sidebar → product grid → payment modal). No API contracts, store interfaces, or user-visible behaviour change — this is a pure visual-layer pass. Each task is self-contained and commits independently.

**Tech Stack:** Next.js 16, React 19, Tailwind CSS v4, V2 design tokens (`surface-*`, `text-*`, `border-*`, `success/warning/danger` semantic tokens, `accent-*`), `Skeleton` primitive (`shape="kpi"|"row"|"card"`), `EmptyState` primitive.

| `border-green-*` | `border-success-border` |

| `bg-orange-*`, `text-orange-*`, `border-orange-*` | `bg-warning-bg-soft`, `text-warning-fg`, `border-warning-border` |
| `bg-red-*`, `text-red-*` (danger) | `bg-danger-bg-soft`, `text-danger-fg` |
| `border-black` / `hover:bg-black` (accent intent) | `border-accent-default` / `hover:bg-accent hover:text-accent-fg` |
| `bg-blue-*`, `text-blue-*` | `text-text-muted` (icons) or `text-text-primary` (prices) |
